form 태그 / input 태그

form 태그

    - 입력 양식 작성하는 판(영역)
    - 내부에 작성된 입력 값 지정된 곳으로 제출하는 역할

    [속성]
    1) action : 제출될 곳 지정하는 속성(주소 작성)
    2) method(방식) : 설정된 곳으로 데이터를 전달하는 방식
    3) name : 이름 지정하는 속성(JS 사용)
    4) target : 제출된 페이지 현재/새 탭으로 열기

input 태그

    - 웹 문서에서 사용자(client)가 입력할 수 있는 양식 제공하는 태그
    - 여러 type 존재, input 의 변형 형태도 다수 존재 

    * input 태그에는 name 속성이 필수 (80% 이상 작성하는 경우 多)

날짜/시간 관련 input 태그

radio, checkbox

    - 여러 연관된 값을 묶어서 선택하는 경우에 사용

    - radio : 여러 값 중 하나 선택
              ex) 성별, 나이대 ...

    - checkbox : 여러 값 중 원하는 모든 것 선택
              ex) 상세검색, 관심분야, 취미 ...

    
    연관된 radio, checkbox는 같은 name 속성 값 가져야 한다

label 태그

    - 입력 관련 태그(input, select, textarea,...)의 제목을 나타내는 태그

    - 사용법 1 : label 태그 내부에 입력 태그를 하나 작성하면 label 태그를 클릭해도 입력 태그 선택됨.
    
    - 사용법 2 : 입력 태그의 id 속성값을 label 의 for 속성 값에 똑같이 적으면 연결됨


select / option 태그

    - select : dropdown. 클릭하면 밑으로 선택창 나옴

    - option : select 에 보여질 메뉴 작성하는 태그
    
    * select와 option은 input의 name, value가 분리된 태그로 생각해

textarea

    - 글 박스. 여러 줄 입력가능

    [속성]
    - name : 제출되는 곳에서 데이터 읽을 때 필요한 이름
    - rows : 행 크기 지정 (rows="5" -> 5행)
    - cols : 열 크기 지정 (cols="50" -> 50열)   == 5행 50열 250자 작성
    - style="resize:none" : 크기 변경 불가

    * textarea는 pre 처럼 작성된 모양 그대로 출력하는 성질

기타 입력 관련 태그

    - type="color" : 색 지정 가능한 입력 상자 출력
    - type="file" : 파일 선택 가능한 탐색기 실행
    - type="hidden" : 숨김. 화면에 안 보이게
                      (숨기고 싶은 값 있을 때 사용. 민증번호 등)


fieldset / legend

    - fieldset : 테두리 만들어서 영역 설정.
                 주로 form 태그 내부에서 입력 요소 그룹 지을 때 사용

    - legend(제목) : fieldset 테두리에 이름 지정 태그

button 태그

    - type 속성 값에 따라 동작이 달라짐

    type="submit" : 제출 버튼 (button 기본 값. type 안 쓰면 자동 submit)
    type="reset" : 버튼 포함 된 form 태그의 모든 입력 값 초기화
    type="button" : 아무런 기능이 없는 버튼
                   -> JS에서 기능을 추가하는 용도의 버튼
